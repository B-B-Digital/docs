# How to request a new feature or page

A site rarely stays finished on launch day — new ideas tend to come up over time: a pricing page worth expanding, a form for booking a consultation, a calendar for scheduling appointments, a separate page for a new service. If you already know you're dealing with something new rather than a fix for something that stopped working (that distinction is covered in [Bug or new feature?](../wsparcie/blad-czy-nowa-funkcja.md)), here's the best way to let us know.

## What's worth including in your request

The more clearly you describe what you're after, the faster we can put together a sensible proposal. It helps to mention:

- **what you're trying to achieve** — for example, "I want clients to be able to book an appointment online, without calling"
- **an example, if you have one** — a link to another business's site with something similar makes communication much easier
- **what it connects to** — whether it's a separate page, a change to something existing, or an integration with an external tool you already use

You don't need to describe it in technical language — just explain how you picture it, the way you'd explain it to a friend.

## What happens next

Once you send your request, we check whether it fits within the scope we already agreed on, or whether it goes beyond the original project. If it does — which is completely normal for a new feature — we'll put together a short quote and a rough timeline before starting the work. You'll always know in advance what it will cost and roughly when to expect it.

Have an idea but aren't sure if it's doable? Just send it over — we'll look into it together before you have to make any decision.
