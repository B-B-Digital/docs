# AGENTS.md ‚Äî B&B Digital Client Docs Site

Client-facing help center for B&B Digital, built with Docusaurus in docs-only mode (no blog, no landing page ‚Äî the FAQ/guides live at the site root). Polish is the canonical content language; English is a full i18n mirror.

## Commands

```bash
pnpm start                # dev server, Polish (default locale)
pnpm start --locale en    # dev server, English ‚Äî dev server only ever renders ONE locale at a time
pnpm build                # production build, both locales ‚Üí build/
pnpm serve                # serve the production build/ ‚Äî required to test the locale dropdown
pnpm typecheck             # tsc, no emit
pnpm docusaurus write-translations --locale en   # regenerate theme translation JSON (navbar/footer/sidebar labels)
```

No test framework. CI (`.github/workflows/ci.yml`) runs `pnpm install --frozen-lockfile` then `pnpm build` on every PR and push to `main`.

Deploy (`.github/workflows/deploy.yml`) runs after CI succeeds on `main`: `wrangler deploy` to Cloudflare Workers (static assets only, `wrangler.jsonc`, no `main` script ‚Äî this site has no server code). Requires `CLOUDFLARE_API_TOKEN` + `CLOUDFLARE_ACCOUNT_ID` repo secrets (a separate Cloudflare account from `BB-Digital_Portfolio` ‚Äî see README "Custom domain" section).

Polish content lives in `docs/`, one folder per sidebar category. English is a 1:1 mirror in `i18n/en/docusaurus-plugin-content-docs/current/` ‚Äî **same folder names, same filenames** (the doc ID is path-derived; renaming an English file breaks the pl‚Üîen mapping).

`sidebars.ts` is explicit (not autogenerated) ‚Äî every new doc must be added to its category's `items` array by doc ID (folder/filename, no extension), in the order it should appear.

Every doc's frontmatter sets a **flat root-level `slug`** (e.g. `slug: /czym-jest-cms`, not folder-derived) plus `sidebar_position` and `title`. Cross-links between docs use **file-relative Markdown links ending in `.md`** (`../faq/czym-jest-cms.md`), never absolute slug paths ‚Äî relative links resolve to the correct per-locale slug automatically; absolute ones don't and silently break in English.

## Gotchas

- **`docs/start.md` is hand-maintained in both locales, not autogenerated.** It lists every category with a bullet per doc. Adding a category/doc without updating `start.md` (Polish AND `i18n/en/.../current/start.md`) leaves it undiscoverable from the homepage ‚Äî `pnpm build` won't catch this.
- **Sidebar *category* labels need a separate i18n translation entry.** Doc titles auto-translate via each file's own frontmatter, but a category's `label` string in `sidebars.ts` only translates in the English UI if a matching key exists in `i18n/en/docusaurus-plugin-content-docs/current.json` (`"sidebar.docs.category.<Polish label>": {"message": "<English label>"}`). Miss this and the English sidebar silently shows Polish category names ‚Äî build passes regardless.
- **i18n markdown fallback is silent.** A missing English translation doesn't fail the build ‚Äî Docusaurus renders the Polish content under the Polish slug instead. Verify translation completeness by diffing file lists per category folder (pl vs. en) and grepping the English build output for Polish words, not just checking build success.
- **Mermaid diagrams render client-side only.** The SSR'd static HTML in `build/` contains no diagram markup at all (no `mermaid` class, no node text) even when correctly configured ‚Äî the diagram mounts via JS after page load. Checking `build/**/*.html` for diagram content will always look "broken" whether or not it actually is; the only real verification is loading the page in a browser.
- **`tsconfig.json` does not extend `@docusaurus/tsconfig`.** That package's base config sets `compilerOptions.baseUrl`, which TypeScript 7 removed entirely (`TS5102`). `tsconfig.json` inlines the same options minus `baseUrl` instead ‚Äî `paths` still resolves correctly relative to the tsconfig file without it. Don't re-add the `extends` line without checking the installed TypeScript major version first.
- **`wrangler.jsonc` has no `main` script on purpose.** This site is pure static output (`assets.directory: ./build`, no dynamic routes) ‚Äî an asset-only Worker needs no JS entrypoint.

## Branding

Colors, fonts, and logo match `docs/branding/brand-guide.html` in `BB-Digital_Portfolio`: primary `#0c7e93` (light mode) / secondary `#22c3da` (dark mode) in `src/css/custom.css`, Space Grotesk (headings) + Inter (body) via Google Fonts, light/dark logo swap (`logo`/`logo.srcDark` in `docusaurus.config.ts`) using `static/img/logo-light.svg` (black mark) and `logo-dark.svg` (white mark).
