# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

Client-facing help center for B&B Digital (Docusaurus, docs-only mode — no blog, no landing page, docs live at the site root). Polish is canonical; English is a full i18n mirror. Read `AGENTS.md` for the stack/commands/structure reference — this file covers the authoring conventions AGENTS.md doesn't.

---

## Commands

```bash
pnpm start                # dev server, Polish — dev server only ever renders ONE locale
pnpm start --locale en    # dev server, English
pnpm build                # both locales → build/, zero errors required
pnpm serve                # serve build/ — required to test the locale dropdown (dev server can't)
pnpm typecheck
```

No test framework. CI runs `pnpm build` only; deploy (Cloudflare Workers via `wrangler deploy`) runs after CI succeeds on `main` — see `AGENTS.md` for the full pipeline.

## Content model

Every doc lives in `docs/<category>/<file>.md` with a 1:1 English mirror at the exact same relative path under `i18n/en/docusaurus-plugin-content-docs/current/<category>/<file>.md` — same filename always (the doc ID is path-derived, so renaming breaks the pl↔en link). `sidebars.ts` is hand-written, not autogenerated: every new doc needs an entry added to its category's `items` array.

Adding a new doc:
1. Create the Polish file with frontmatter (`slug: /<flat-slug>`, `sidebar_position`, `title`).
2. Add its ID to `sidebars.ts`.
3. Copy to the English mirror path, same filename, translate content + override `slug`/`title`.
4. Cross-link using file-relative `.md` links (`../faq/czym-jest-cms.md`) — never absolute slugs (`/czym-jest-cms`). Relative links resolve to the correct per-locale slug automatically; absolute ones silently break in English.
5. If it's the first doc in a brand-new category, also add its category's label translation to `i18n/en/docusaurus-plugin-content-docs/current.json` (`sidebar.docs.category.<Polish label>`) and add matching bullets to `docs/start.md` **and** its English mirror — neither is autogenerated.
6. `pnpm build` must pass for both locales before considering it done — but build success alone doesn't prove the English translation is complete (see Gotchas) or that a Mermaid diagram actually renders (also see Gotchas).

## Voice & style — every article must follow this

The audience is non-technical Polish small-business owners (dentists, lawyers, coaches) who've never bought a website before. The existing articles in `docs/faq/` and `docs/wspolpraca/` are the tone reference — read a couple before writing anything new.

- Direct "Ty/Twoja" address, never "Państwo" or third-person distancing. The agency speaks as "my" (we), never a faceless "system"/"platforma".
- Written as if from a real person at the agency — no bot-like sign-offs ("Zespół B&B Digital" etc.).
- Vary sentence length; no walls of bullets (prose lead-in before any list, bullets only for genuinely enumerable content); no heading that just restates itself ("Wprowadzenie", "Podsumowanie"); end with a concrete forward link or one-line reassurance, never a generic recap.
- Concrete real-world analogies in the same register as existing ones (domain = address, hosting = building, SSL = padlock) — invent new ones in that spirit, don't force them.
- **Banned AI-tell phrases** (or close paraphrases): "W dzisiejszych czasach", "W erze cyfrowej", "Zanurzmy się", "Odblokuj", "Rewolucyjny", "Kompleksowe rozwiązanie", "Bezproblemowo", "Warto zaznaczyć, że...", "Podsumowując" as an opener. No rule-of-three adjective stacking ("szybko, łatwo i wygodnie"). Grep new content for these before considering it done.
- When writing a set of similar articles (e.g. one per CMS content type), deliberately vary the opening/structure of each — identical paragraph shapes across a category read as machine-generated.
- Bold sparingly (one key phrase, never whole sentences); no emoji; no exclamation-heavy marketing copy.

## Gotchas

- **`pnpm build` passing is not sufficient verification.** Three known ways this repo looks fine but isn't: (1) Mermaid diagrams render client-side only — the static `build/` HTML has zero diagram markup even when working correctly, so grepping build output tells you nothing; verify in an actual browser. (2) A missing English translation doesn't fail the build, it silently falls back to Polish content under the Polish slug — diff file lists per category (pl vs. en) and grep the English build output for Polish words. (3) A new sidebar category's label won't translate without a separate JSON entry (see Content model step 5) — build passes regardless.
- **`tsconfig.json` intentionally does not extend `@docusaurus/tsconfig`** — that package sets `compilerOptions.baseUrl`, which TypeScript 7 removed (`TS5102`). Don't re-add the `extends` unless you've confirmed the installed TypeScript major version supports it again.
- **`wrangler.jsonc` has no `main` script on purpose** — this is a static-assets-only Worker (`assets.directory: ./build`), Docusaurus's output needs no server code.
